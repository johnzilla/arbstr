<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> response headers | complexity_score and tier are server-generated, not user-input |
| DB columns | New columns are nullable, written by server only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Information Disclosure | x-arbstr-complexity-score header | accept | Score is derived from user's own request; revealing it helps debugging, not an attacker. No PII. |
| T-20-02 | Information Disclosure | x-arbstr-tier header | accept | Tier is a coarse routing label (local/standard/frontier); does not reveal provider internals. |
| T-20-03 | Injection | tier column in SQL | mitigate | Tier values come from Tier::Display (enum with 3 fixed strings), not user input. No SQL injection possible. |
| T-20-04 | Denial of Service | format!("{:.3}", score) | accept | Score is f64 from scorer, not user-controlled size. Fixed 3dp format string cannot produce unbounded output. |
</threat_model>
