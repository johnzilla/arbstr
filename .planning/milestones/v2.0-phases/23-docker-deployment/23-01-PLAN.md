<task type="auto">
  <name>Task 1: Generate sqlx offline metadata and create Dockerfile</name>
  <files>Dockerfile, .dockerignore, .sqlx/, .gitignore</files>
  <read_first>
    - Cargo.toml (sqlx features and version)
    - .gitignore (to add .sqlx/ exception)
    - src/main.rs (binary entry point name)
    - migrations/ (list migration files to confirm they exist)
  </read_first>
  <action>
    1. Install sqlx-cli if not present: `cargo install sqlx-cli --no-default-features --features sqlite`
    2. Create a temporary SQLite database and run migrations so sqlx can introspect queries:
       ```
       DATABASE_URL=sqlite:./tmp-prepare.db cargo sqlx database create
       DATABASE_URL=sqlite:./tmp-prepare.db cargo sqlx migrate run
       ```
    3. Generate offline metadata: `DATABASE_URL=sqlite:./tmp-prepare.db cargo sqlx prepare --workspace`
       This creates the `.sqlx/` directory with JSON query metadata files.
    4. Clean up: `rm -f tmp-prepare.db tmp-prepare.db-shm tmp-prepare.db-wal`
    5. Update `.gitignore` to NOT ignore `.sqlx/` — add `!.sqlx/` after the database ignore rules so the offline metadata is committed.

    6. Create `.dockerignore` with these entries:
       ```
       target/
       .git/
       .planning/
       .github/
       *.db
       *.db-shm
       *.db-wal
       .env
       .env.*
       config.toml
       ```

    7. Create `Dockerfile` (per D-03, D-04, D-05):
       ```dockerfile
       # Stage 1: Build
       FROM rust:1.86-slim AS builder
       RUN apt-get update && apt-get install -y pkg-config libssl-dev && rm -rf /var/lib/apt/lists/*
       WORKDIR /app
       COPY Cargo.toml Cargo.lock ./
       COPY src/ src/
       COPY migrations/ migrations/
       COPY .sqlx/ .sqlx/
       ENV SQLX_OFFLINE=true
       RUN cargo build --release

       # Stage 2: Runtime
       FROM debian:bookworm-slim
       RUN apt-get update && apt-get install -y --no-install-recommends ca-certificates curl && rm -rf /var/lib/apt/lists/*
       COPY --from=builder /app/target/release/arbstr /usr/local/bin/arbstr
       EXPOSE 8080
       ENTRYPOINT ["arbstr"]
       CMD ["serve", "-c", "/config/config.toml"]
       ```

       Key details per decisions:
       - Per D-03: rust:1.86-slim builder, debian:bookworm-slim runtime
       - Per D-04: SQLX_OFFLINE=true, .sqlx/ copied into build context
       - Per D-05: No multi-arch, amd64 only
       - Install ca-certificates (for HTTPS to providers) and curl (for healthcheck) in runtime
       - Copy only necessary files (not full COPY . .) to leverage Docker layer caching
       - EXPOSE 8080 matches server.listen default
  </action>
