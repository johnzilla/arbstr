    **2. Create src/proxy/discovery.rs with discover_models function (per D-01, D-02, D-03, D-06):**

    ```rust
    use crate::config::ProviderConfig;
    use reqwest::Client;
    use std::time::Duration;

    #[derive(serde::Deserialize)]
    struct ModelsResponse {
        data: Vec<ModelEntry>,
    }

    #[derive(serde::Deserialize)]
    struct ModelEntry {
        id: String,
    }

    /// Discover models for providers with auto_discover enabled.
    /// Called once during server startup (D-01: no periodic refresh).
    /// On success, replaces provider.models with discovered ids (D-03, D-06: exact names).
    /// On failure, logs warning and keeps static models (D-02: non-blocking startup).
    pub async fn discover_models(providers: &mut [ProviderConfig], client: &Client) {
        for provider in providers.iter_mut() {
            if !provider.auto_discover {
                continue;
            }

            let url = format!("{}/models", provider.url.trim_end_matches('/'));
            tracing::info!(provider = %provider.name, url = %url, "Discovering models");

            match client
                .get(&url)
                .timeout(Duration::from_secs(5))
                .send()
                .await
            {
                Ok(resp) if resp.status().is_success() => {
                    match resp.json::<ModelsResponse>().await {
                        Ok(models_resp) => {
                            let model_ids: Vec<String> =
                                models_resp.data.into_iter().map(|m| m.id).collect();
                            tracing::info!(
                                provider = %provider.name,
                                models = ?model_ids,
                                count = model_ids.len(),
                                "Discovered models"
                            );
                            provider.models = model_ids;
                        }
                        Err(e) => {
                            tracing::warn!(
                                provider = %provider.name,
                                error = %e,
                                "Failed to parse /v1/models response, keeping static models"
                            );
                        }
                    }
                }
                Ok(resp) => {
                    tracing::warn!(
                        provider = %provider.name,
                        status = %resp.status(),
                        "Discovery endpoint returned non-success status, keeping static models"
                    );
                }
                Err(e) => {
                    tracing::warn!(
                        provider = %provider.name,
                        error = %e,
                        "Failed to reach discovery endpoint, keeping static models"
                    );
                    if provider.models.is_empty() {
                        tracing::warn!(
                            provider = %provider.name,
                            "Provider has no models after failed discovery -- it won't match any requests until restarted with the provider available"
                        );
                    }
                }
            }
        }
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbstr core -> provider /v1/models | HTTP GET to operator-configured URL during startup |
| /v1/models response -> ProviderConfig.models | Untrusted JSON parsed into model list |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Denial of Service | discover_models | mitigate | Per-request 5-second timeout via `reqwest::Client::get().timeout(Duration::from_secs(5))` prevents slow/hung endpoints from blocking startup |
| T-24-02 | Information Disclosure | discover_models SSRF | accept | Provider URLs come from operator-controlled config.toml (not user input). Operator already controls all provider URLs. No additional SSRF surface vs existing inference forwarding. |
| T-24-03 | Tampering | Malicious /v1/models response | accept | Model names are strings used only for matching in the router. A malicious model name has no code execution path -- worst case is a confusing model name in /v1/models output. Operator controls which providers are configured. |
| T-24-04 | Denial of Service | Oversized /v1/models response | mitigate | reqwest body size is bounded by the 5-second timeout. A response with thousands of models parses into Vec<String> which is bounded by available memory. For a localhost service this is negligible risk. Serde deserialization rejects malformed JSON. |
</threat_model>
