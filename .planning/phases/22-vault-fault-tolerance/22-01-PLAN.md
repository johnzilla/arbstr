<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbstr -> vault | Vault may be unreachable; pending settlements bridge this gap |
| SQLite -> reconciliation | Persisted data replayed automatically on restart |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Tampering | pending_settlements table | accept | SQLite is local, same trust boundary as arbstr process. No external write access. |
| T-22-02 | Denial of Service | reconcile_once | mitigate | Eviction after 10 attempts prevents unbounded growth of pending_settlements table (per D-03). Backpressure flag already blocks new requests when pending count exceeds threshold. |
| T-22-03 | Information Disclosure | tracing::error eviction log | accept | Logs contain reservation_id (opaque UUID) and settlement type, no secrets or PII. Standard operational logging. |
| T-22-04 | Repudiation | evicted settlement | accept | Per D-05, no dead-letter table. Eviction is logged at error level for operational alerting. Acceptable for v2.0 -- dead-letter can be added if eviction rate is high in production. |
</threat_model>
